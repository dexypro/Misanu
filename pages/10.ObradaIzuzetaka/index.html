Zadatak 340: (X_Exceptions.X_FinallyBlock)

Definisati klasu finallyBlockWithException i u njoj:
- statičku metodu procA koja simulira RuntimeException i sadrži try, catch i finally block.
- statičku metodu procB koja sadrži try, catch i finally block.
- statičku metodu procC koja sadrži try, catch i finally block.

Definisati pokretačku klasu Main i u njoj:
- kreirati objekat klase finallyBlockWithException pod nazivom fbwe.
- u try / catch bloku pokrenuti metodu procA objekta fbwe.
- pokrenuti ostale dve metode procB i procC objekta fbwe.

Klasa:

package X_Exceptions.X_FinallyBlock;

public class finallyBlockWithException {

  // Throught an exception out of the method

    static void procA() {
      try {
          System.out.println("inside procA");
          throw new RuntimeException("demo"); // throws je uzglavlju metode
          // throw koriste se za bacanje izuzteaka u telu metode
      } finally { /* finally block se izvrasav bez obzira da li je ili nije doslo do izuzetka*/
          System.out.println("procA");
      }
    }

    static void procB() {
      try {
          System.out.println("procB");
          return; /* break, continue, System.exit(0) */
      } finally {
          System.out.println("procB's finally");
      }
    }

    // Execute a try block normally.
    static void procC() {
      try {
          System.out.println("inside procC");
      } finally {
          System.out.println("procC's finally");
      }
    }
}

Pokretačka klasa:

package X_Exceptions.X_FinallyBlock;

public class Main {

    public static void main(String[] args) {

      finallyBlockWithException fbwe = new finallyBlockWithException();

      try {
          fbwe.procA();
      } catch (Exception e) {
          System.out.println("Exception caught");
      }

      fbwe.procB();
      fbwe.procC();

    }

}

inside procA
procA
Exception caught
procB
procB's finally
inside procC
